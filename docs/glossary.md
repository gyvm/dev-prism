# 用語集（Glossary）

凍結レポート（Dev Prism）の設計・議論で使う用語の定義。曖昧さを避けるため、grilling で確定した意味を記録する。関連: [ADR 0001](adr/0001-frozen-report-information-architecture.md)。

| 用語 | 定義 |
|---|---|
| **凍結レポート** | 週次に焼き付ける自己完結 HTML（クライアント JS ゼロ）。データは凍結、描画のみ rerender 可能（方式D）。 |
| **問い別 IA** | セクションを描画タイプ（カード/AI/チャート）ではなく問い（数字 → 理由 → 会話）で構成する情報設計。原因と結果を同居させる。 |
| **体温（フロー）** | チームの開発の流れの状態。週の入口で集計リードタイム等の数字として掴む対象。良し悪しの断定ではなく状態の把握。 |
| **変更リードタイム** | PR の作成 → マージまでの時間。中央値で集計。本レポートの一次指標。 |
| **デプロイ頻度** | 本レポートでは **デフォルトブランチ（master 等）へのマージ数**。リリース自動化は組織により異なるためマージ数を正直な代理値とする。デプロイそのものではない。 |
| **変更失敗率 / MTTR** | `Revert "…"` という GitHub 既定の PR タイトルで巻き戻しを自動検知して算出（ラベル運用に依存しない初期実装）。 |
| **前週比** | 数値の対先週変化。体温の「変化」を表す。v0.1 の基準線。 |
| **目標比** | 数値の対目標達成度。「異常（遅すぎる等）」の検出に必要。v0.1 ではスコープ外（後回し）。 |
| **その数字に効いた PR** | *今週マージ/クローズされた* 中でリードタイムの長い PR。数字（過去形）の理由を説明する。closed が対象。 |
| **来週確認したいこと** | *まだ開いている* PR（長く放置 / 未解決レビュー / コメント後に停止）。未来のアクション。open が対象。「その数字に効いた PR」との分割線は open/closed。 |
| **数値目標に対するアクション** | 数字（効率化・目標見直し）から導く打ち手。①開発メトリクスの締め。 |
| **気づきに対するアクション** | 数値と無関係な振り返りの気づき・成果物から導く打ち手。②開発内容の要約の締め。上記とは別ジョブ。 |
| **貢献の棚卸し** | 見落とし防止・小さな貢献の網羅。体温（フロー健全性）とは別ジョブとして扱う。 |
| **args ファースト** | 設定を config.toml ではなく args / GitHub Action `with:` 入力で渡す方針。将来の目標値もここに置く（[product-direction-config メモリ参照]）。 |
| **レジストリ（カタログ）** | 分析「部品」が何存在するかだけを持つ明示的マップ（`COMPUTE_REGISTRY` / `AI_REGISTRY`）。**順序は持たない**。[ADR 0002](adr/0002-ai-prompt-architecture.md)。 |
| **順序の所在** | レポートの並び順は `render.tsx` の明示配列だけが持つ。compute と AI を跨いで交互配置するためレジストリでは表現できない。並びを変える時は render の配列1箇所を編集。 |
| **プロンプト埋め込み（方式C）** | プロンプトを `src/prompts/*.md` で著し、ビルド時に文字列として `dist` へ焼き込む方式。実行時 FS ゼロ・必ず同梱。生 `.md` を実行時に読む方式Fは配布で黙死するため不採用。 |
| **スキル発見の廃止** | Copilot の `skillDirectories`＋`Use the "X" skill` 間接層を撤去し、プロンプト全文をアプリが直接 SDK に送る。ユーザーによるプロンプト拡張も廃止（built-in 固定）。 |
